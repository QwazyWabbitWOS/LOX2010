R1Q2DED Server Binary Installation ..

Now we are going to configure and compile the newest version (at the time of writing) of R1Q2DED for Linux. This is an awesome server for Quake2 thats based on security, stability and speed. This process is two fold to do it right and make things easier in the long run. First we will build the R1Q2 Updater that fetches the latest version of R1Q2 and lastly we will use R1Q2Updater to get the newest version.

9. Enter into your Quake2 directory, get the files for the R1Q2Updater, then build it: 


# cd /home/q2user/quake2

# wget http://www.r1ch.net/stuff/r1q2/src/update.c

# gcc -O2 -o r1q2updater -lcurl -lz -lcrypto update.c

The resulting file from the above build command should be "r1q2updater". Now with the updater built we can easily grab the latest version and even automate the update process with a cron script if you wanted. Make sure you are in your main Quake2 directory and run r1q2updater. 


# ./r1q2updater

The above command should output something like this: 


[q2user@tombgamer quake2]$ ./r1q2updater

Press ENTER to exit.

10. If all went well you should the latest version of r1q2. This will be the main server executable for running your server. r1q2 has many, many improvements over the older 3.21 server binary and best of all it's an active project in still in development.

If you don't want to run the R1Q2 server binary you can also build the 3.21 server instead, click here. You will need one or the other but I highly recommend R1Q2 over 3.21.
